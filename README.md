# Airline-Crew-Pairing-Optimization
Airline crew pairing is a large-scale set partitioning problem where crews must be assigned to flight legs while satisfying FAA regulations, duty limits, rest requirements, and base constraints. Enumerating all feasible pairings is computationally intractable, so this project implements a Branch-and-Price framework using Column Generation.
# Airline Crew Pairing Optimization using Column Generation

*A scalable implementation of the Airline Crew Pairing Problem using Set Partitioning, Column Generation, and Mixed Integer Programming in Julia.*

---

## Overview

The Airline Crew Pairing Problem (CPP) is one of the largest and most computationally challenging optimization problems in airline operations. Given a schedule of flight legs, the objective is to construct feasible crew pairings that cover every scheduled flight exactly once while minimizing operational costs and satisfying regulatory and operational constraints.

The number of feasible pairings grows exponentially with the size of the flight schedule, making complete enumeration computationally impossible even for moderately sized airline networks. Modern airline optimization systems therefore rely on decomposition techniques rather than explicitly generating every feasible pairing.

This project implements a **Column Generation** framework that dynamically generates only promising crew pairings during optimization. Instead of solving an enormous set partitioning problem containing billions of variables, the algorithm alternates between solving a Restricted Master Problem (RMP) and a Pricing Problem until no improving pairings remain.

The implementation is written in **Julia** using **JuMP** and **Gurobi**, and demonstrates how decomposition techniques dramatically improve computational performance for large-scale airline scheduling problems.

---

## Problem Statement

Given a set of scheduled flights, determine a minimum-cost collection of legal crew pairings such that:

* Every flight is covered exactly once.
* Every pairing begins and ends at the designated crew base.
* Minimum connection times are satisfied.
* Maximum duty time limits are respected.
* Maximum pairing duration constraints are enforced.
* Layover and deadhead costs are minimized.

The resulting optimization model is formulated as a **Set Partitioning Problem**, where every feasible pairing corresponds to a decision variable.

---

## Why Column Generation?

A direct formulation requires generating every feasible crew pairing before optimization.

For realistic airline schedules this quickly becomes impossible:

* Hundreds of flights
* Billions of feasible pairings
* Massive memory requirements
* Extremely long solution times

Column Generation overcomes this combinatorial explosion by generating only those pairings that have the potential to improve the current solution.

Instead of solving one enormous optimization problem, the algorithm repeatedly solves two smaller problems:

1. A **Restricted Master Problem (RMP)** that selects the best pairings currently available.
2. A **Pricing Problem** that searches for new pairings with negative reduced cost.

The process terminates once no improving pairing can be found, guaranteeing optimality of the LP relaxation.

---

### Mathematical Formulation

#### Sets

* (F): Set of all scheduled flight legs.
* (P): Set of all feasible crew pairings.

#### Parameters

* (c_p): Operating cost of pairing (p).
* (a_{fp}): Binary parameter equal to 1 if flight (f) is included in pairing (p), and 0 otherwise.

#### Decision Variable

$$
y_p =
\begin{cases}
1, & \text{if pairing } p \text{ is selected},\
0, & \text{otherwise}.
\end{cases}
$$

#### Objective

Minimize the total operating cost of the selected crew pairings:

$$
\min \sum_{p\in P} c_p y_p
$$

where (c_p) includes layover costs and deadhead repositioning costs.

---

### Constraints

#### Flight Coverage

Every scheduled flight must be covered by exactly one crew pairing.

$$
\sum_{p\in P} a_{fp}y_p = 1,
\qquad \forall f\in F
$$

---

#### Binary Decision Variables

A pairing is either selected or not selected.

$$
y_p \in {0,1},
\qquad \forall p\in P
$$

---

### Pricing Problem Constraints

During column generation, new pairings are generated by solving a reduced-cost shortest-path Mixed Integer Program over the flight network.

The pricing model satisfies the following constraints.

#### Source Constraint

Exactly one arc leaves the crew base.

$$
\sum_{j} x_{sj}=1
$$

---

#### Sink Constraint

Exactly one arc enters the crew base.

$$
\sum_i x_{it}=1
$$

---

#### Flow Conservation

Every intermediate flight entered must also be exited.

$$
\sum_i x_{ij} = \sum_k x_{jk},
\qquad \forall j\in F
$$

---

#### Maximum Duty Time

The total flying time and deadhead time within a duty period cannot exceed 16 hours.

$$
\sum_{(i,j)} \text{DutyTime}*{ij},x*{ij} \le 16
$$

---

#### Maximum Pairing Duration

The elapsed time from report to release for a pairing cannot exceed 24 hours.

$$
\text{PairingDuration} \le 24
$$

---

#### Minimum Connection Time

A connection between two flights is feasible only if the ground time is at least one hour.

$$
\text{Departure}_j - \text{Arrival}_i \ge 1
$$

---

#### Crew Base Constraint

Every generated pairing must begin and terminate at the designated crew base.

---

#### Feasible Connection Constraint

An arc between two flights exists only if:

* the arrival airport of flight (i) matches the departure airport of flight (j), or
* a legal deadhead reposition is possible, and
* sufficient connection time exists.

---

### Reduced Cost

After solving the Restricted Master Problem, dual prices ($$\pi_f$$) are obtained for each flight coverage constraint.

The pricing problem minimizes the reduced cost

$$
\bar c_p = c_p - \sum_{f\in F}\pi_f a_{fp}
$$

A newly generated pairing is added to the Restricted Master Problem whenever

$$
\bar c_p < 0
$$

Column generation terminates when no pairing with negative reduced cost exists.


---

## Solution Methodology

The implementation follows the classical Column Generation framework.

1. Generate an initial feasible set of single-flight pairings.
2. Solve the Restricted Master Problem.
3. Extract dual prices from the flight coverage constraints.
4. Solve a reduced-cost shortest-path Mixed Integer Program.
5. Add every negative reduced-cost pairing discovered.
6. Repeat until no improving pairings remain.
7. Solve the final integer master problem using the generated columns.

Unlike traditional implementations that add only one pairing per iteration, this project also implements **multi-column pricing** using Gurobi's Solution Pool to generate multiple improving pairings simultaneously, substantially reducing the number of master problem iterations.

## Results

Three synthetic hub-and-spoke airline instances were evaluated.

| Airports | Flights | Traditional Pricing | Multi-Column Pricing |
| -------- | ------- | ------------------: | -------------------: |
| 6        | 50      |             2.754 s |              0.006 s |
| 8        | 100     |            30.720 s |              0.011 s |
| 12       | 150     |           148.352 s |              0.028 s |

Multi-column pricing dramatically reduced the number of Column Generation iterations.

| Flights | Single Column | Multi Column |
| ------- | ------------: | -----------: |
| 50      |            37 |            3 |
| 100     |            78 |            6 |
| 150     |           161 |            6 |

These results demonstrate the effectiveness of generating multiple improving pairings during each pricing iteration rather than adding only the single best column.
